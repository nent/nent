Partial Content

Optimize your HTML app, by leveraging lazy-loading and only pulling in heavy dependencies when you absolutely need them.

Include Remote HTML

To make big applications manageable, developers adhere to a strict 'do not repeat yourself' principle. If the same HTML is exists in more than one place, you'll want to manage it in a singe place.

It makes updates much easier, and you can leverage its lazy-loading feature to make the site render faster.

Organize these fragments however you like, then inject them into your application, using the <n-content> component.

Just specify the source location and the element does everything else. If the file is not on the same server as the master page, you may have to configure CORS. You can read more about that in the element docs.

If the fragment has script or css-dependencies use the <n-content-reference> element to ensure the dependencies are loaded. Read more about this below.

Guarantee Dependency References

Once you start breaking your site up into fragments, you'll find it improves performance to not pull in any remote scripts or style-sheets until you need them. Using the <n-content-reference> component, you decorate fragments with the remote dependencies that HTML needs, and the element will make sure it's there. If the user already went to a page-title with that component, it won't be added again. Keep your HTML light and functional.
