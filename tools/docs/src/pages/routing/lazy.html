Lazy Loaded Routes

HTML when you need it, not when you don't.

A route's content could be inline or fetched from a remote file as needed. This is called "lazy-loaded".

You get the benefit of shared layouts and elements without the need for an application-server, client-side framework or static-site generation process.

How it Works

Instead of wrapping content, you can specify a remote file to fetch with both <n-view> and <n-view-prompt> elements. The referenced file won't be fetched until the URL path matches the path set in the url attribute.

This gives you the the ability to keep the initial page-size light and only fetch heavier HTML fragments when needed.

Lazy Route Example

Include a content-src attribute in the <n-view> or <n-view-prompt> elements to "lazy-load" the inner-html when the route is activated.

TIP: A mixture of inline and remote content can be used within the same route. Be sure to include the slot="content" attribute for content that should not show on child-routes.

Lazy Content

There are other ways to lazy-load content. You can use the <n-content> or <n-content-markdown> elements along with their defer-load attribute to essentially do the same thing.

When routes are activated they remove the defer-load attribute from child elements, which in-turn performs a fetch on route-activate.
